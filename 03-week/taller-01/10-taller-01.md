## Qué hay que hacer?

::contents::
A partir de las **imágenes de referencia** que se les suministran, deben recrear una página web completa usando solo **HTML y CSS**.

No se permite usar frameworks ni librerías externas (Bootstrap, Tailwind, etc.).

La página debe verse bien en **móvil y en escritorio**.

## Secciones de la página

::contents::
La página tiene las siguientes secciones — cada una es un bloque a construir:

1. **Navbar** — barra de navegación fija
2. **Hero** — sección principal con título y botón
3. **Nosotros** — texto descriptivo
4. **Servicios** — tarjetas de servicios
5. **Portafolio** — grilla de proyectos
6. **Contacto** — formulario
7. **Footer**

## Requisitos técnicos

::contents::
✅ HTML semántico (`<nav>`, `<section>`, `<footer>`, etc.)

✅ CSS propio — sin frameworks

✅ Flexbox o Grid para los layouts

✅ Responsive: se ve bien en móvil **y** en escritorio

✅ Un solo archivo `.html` con el `<style>` incluido

❌ No se acepta CSS copiado directamente de la referencia
